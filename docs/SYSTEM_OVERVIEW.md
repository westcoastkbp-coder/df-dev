# Digital Foreman — System Overview

## 1. System Identity

Digital Foreman is an Execution OS.

It is NOT:
- chatbot
- assistant
- CRM
- workflow tool

It IS:
- deterministic execution system
- control layer over tools and actions
- state-driven operational system

Core flow:

intent → task → action → verification → state update

---

## 2. Product Model

Digital Foreman exists as one system with multiple deployment forms:

### BusinessBox
- business execution system
- handles intake (phone / SMS / email / operator)
- executes real business workflows

### OwnerBox
- upper-level control system
- provides owner-level visibility and control
- operates above business execution layer

### West Coast KBP
- real-world proof environment
- validates system behavior in live operations

---

## 3. System Principle

Digital Foreman is not an interface.

Interfaces (UI, voice, apps) are only surfaces.

Core value:
- execution
- control
- verification
- state continuity

---

## 4. Source of Truth

System truth lives in:

- repository
- architecture documents
- state files

NOT in:
- chat
- temporary memory
- UI
